# 内置曲谱库

网页工作台会在构建时自动收集 `src/library/**/*.jabc`。曲谱库是随 GitHub Pages 发布的只读内容：网页不保存 GitHub 令牌，也不会从浏览器直接修改仓库。

## 目录约定

```text
src/library/
  儿歌/
    two-tigers.jabc
  练习曲/
    scale-study.jabc
```

- 目录名是网页中的分类；嵌套目录使用 ` / ` 连接显示。
- 相对路径去掉 `.jabc` 后是稳定曲谱 ID。
- `T:` 显示为标题；缺少时使用文件名。`C:` 显示为作者或来源。
- 每首曲谱必须是可由 `parseJabc` 成功解析的完整 JABC 文本。

## 新增曲谱

1. 在合适分类目录下新建 ASCII 文件名的 `.jabc` 文件。
2. 补全 `T:`、`C:`、`M:`、`L:`、`Q:` 和 `K:` 等曲谱头。
3. 运行 `npm test`、`npm run typecheck` 和 `npm run build`。
4. 提交文件并推送 `main`；GitHub Actions 会自动部署更新后的曲谱库。

不需要维护额外清单。Vite `import.meta.glob` 会自动发现新文件。CI 测试会解析全部内置曲谱；网页运行时也会隔离无效文件，在曲谱栏显示路径和解析错误，其他曲谱仍可正常使用。

## 网页行为

曲谱栏支持标题、作者、分类和 ID 子串搜索，并可按分类筛选。载入曲谱会停止当前播放并刷新编辑器、预览和导出结果。若编辑内容已偏离载入时的原文，切换曲谱前会请求确认。
