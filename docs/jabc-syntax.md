# JABC 语法规范

本文档描述当前 `0.1.x` 解析器已经实现的 JABC 子集。未列入“当前语法”的记号均视为不支持，解析器会返回结构化错误。

## 文本与注释

- 输入按行解析，支持 LF 和 CRLF 换行。
- 空行被忽略。
- `%` 开始行注释；其后的内容不会参与解析。
- 音乐 token 可用空白分隔，小节线 `|` 无需与相邻 token 留空格。
- 音乐正文的物理换行是谱面系统换行；同一行中的小节在 ABC、MusicXML、简谱和五线谱中保持同一系统。

```abc
% 完整行注释
K:C jianpu % 行内注释
|1 2 3 1|
```

## 头部字段

字段格式为单个 ASCII 字母、冒号和值：`F:value`。

| 字段 | 含义 | 示例 |
| --- | --- | --- |
| `X:` | 曲目编号 | `X:1` |
| `T:` | 标题 | `T:两只老虎` |
| `C:` | 作曲者或来源 | `C:传统儿歌` |
| `M:` | 拍号，不约分 | `M:4/4` |
| `L:` | 默认音符时值 | `L:1/4` |
| `Q:` | 速度 | `Q:1/4=120` |
| `K:` | 简谱主音及可选调式 | `K:D jianpu` |
| `V:` | 切换或声明声部 | `V:melody` |
| `w:` | 当前声部、前一条音乐源码行的空白分隔歌词 | `w: 两 只 老 虎` |

`K:` 的最后一项必须是 `jianpu`。当前主音支持 `C`、`C#`、`Db`、`D`、`D#`、`Eb`、`E`、`F`、`F#`、`Gb`、`G`、`G#`、`Ab`、`A`、`A#`、`Bb` 和 `B`。例如 `K:A minor jianpu` 会将 `minor` 保存为调式，但当前阶段不解释其音程语义。

`V:` 会结束当前声部正在收集的小节，并切换到指定声部。声部 ID 是 `V:` 后的第一个非空白 token；后续属性暂不解释。音乐正文中也可以使用 inline voice marker，例如 `[V:melody]`。

音乐正文中可以使用 inline key change，例如 `[K:G jianpu]`。该事件不占时值，从当前位置开始把后续音级按新的 `1=G` 映射；ABC、MusicXML、播放和渲染都会沿用新的当前调，直到再次遇到 key change。

其他字段按出现顺序保存在 `header.extraFields`，不参与音乐语义。

## 多声部

```abc
K:C jianpu
V:melody
| 1 2 3 4 |
w: 高 声 部 唱
V:bass
| 1, 3, 5, 1 |
w: 低 声 部 唱
```

等价地，也可以在正文中切换声部：

```abc
K:C jianpu
[V:melody] | 1 2 3 4 |
[V:bass] | 1, 3, 5, 1 |
```

每个声部拥有独立的小节、事件和歌词。未出现 `V:` 时，解析器使用 `default` 声部。

## 音乐正文

```text
body        = { voice | key-change | repeat-marker | tuplet | note | rest | extension | barline } ;
voice       = "[V:", voice-id, "]" ;
key-change  = "[K:", key-field-value, "]" ;
repeat-marker = "!segno!" | "!coda!" | "!fine!" | "!D.C.!" | "!D.S.!" | "!dacapo!" | "!dacoda!" ;
tuplet      = "(3" ;
note        = [ slur-start ], [ tie-end ], [ accidental ], degree, [ octave ], [ duration ], [ dots ], [ tie-start ], [ slur-end ] ;
rest        = ( "0" | "z" ), [ duration ], [ dots ] ;
accidental  = "#" | "##" | "b" | "bb" | "=" ;
degree      = "1" | "2" | "3" | "4" | "5" | "6" | "7" ;
octave      = "'", { "'" } | ",", { "," } ;
duration    = "/", [ positive-integer ] | "*", positive-integer | "w" | "h" | "q" | "e" | "s" ;
dots        = "." | ".." ;
slur-start  = "(" ;
slur-end    = ")" ;
tie-start   = "~" ;
tie-end     = "~" ;
extension   = "-" ;
barline     = "|" | "||" | "|]" | "[|" | "|:" | ":|" ;
ending      = "[", positive-integer ;
```

修饰符必须按“变音、音级、八度、时值、附点”的顺序书写，例如 `#4'e.` 或 `#4'/2.`。各规则如下：

- `[K:G jianpu]` 在正文中改变后续音符的当前调，不占时值，也不消耗歌词。
- `!segno!`、`!coda!`、`!fine!`、`!D.C.!`、`!D.S.!`、`!dacapo!`、`!dacoda!` 是反复导航标记，不占时值，也不消耗歌词。简谱中 `!segno!` / `!coda!` 显示在小节线正上方，`!D.C.!` / `!D.S.!` / `!fine!` 显示在小节线正下方；ABC 导出会保留这些装饰。播放层会先展开本地反复，再在遇到 `!D.C.!` / `!D.S.!` 时分别回到曲首或 `!segno!`，并从回跳目标按事件顺序扫描：`!fine!` 和可跳转的 `!coda!` 谁先出现谁生效；若 `!coda!` 之前没有位于 D.C./D.S. 之后的目标 Coda 段，则继续寻找 `!fine!` 或播到曲尾。
- `1'`、`1''` 分别升高一个、两个八度；`1,`、`1,,` 同理降低。
- `#4`、`b7`、`=3` 表示升、降、还原；`##4`、`bb7` 表示双升、双降。
- `w`、`h`、`q`、`e`、`s` 是绝对时值后缀，分别表示全音符、二分音符、四分音符、八分音符、十六分音符；例如 `1e` 永远是八分音符，不受 `L:` 影响。
- `/` 等同 `/2`；`1/4` 是默认时值的四分之一。
- `1*2`、`1*3` 分别是默认时值的两倍、三倍，倍率必须为正整数。
- `1.` 是原时值的 `3/2`，`1..` 是原时值的 `7/4`；时值修饰可与附点组合，例如 `1e.` 是附点八分音符。
- rest 支持相同的时值和附点后缀，例如 `0e`、`zq.`、`0/2`、`z*2.`。
- `(1` 表示该音开始圆滑线，`3)` 表示该音结束圆滑线；例如 `| (1 2 3) |` 不改变播放时值，只影响显示和导出。
- `1~` 表示该音开始 tie，`~1` 表示该音结束 tie；例如 `| 1~ | ~1 |` 会在播放层合并为一个持续音。
- `(3` 表示后续三个音符或休止符构成三连音，采用“三音占两音”的时值规则；例如 `| (3 1 2 3 |` 中每个四分音会播放为 `1/6` 全音符时值。

未声明 `L:` 时默认使用 `1/4`。`-` 采用一个当前 `L:` 单位并保留为独立事件，由导出或播放层延长前一个音符。行首 `|` 不生成空小节。`|:` 和 `[|` 会作为下一小节的左小节线，`:|`、`||`、`|]` 和普通 `|` 会保存为当前小节的右小节线。`[1`、`[2` 等 ending 标记会附着到后续小节。

歌词按源码音乐行分配：每条 `w:` 对应同一声部里它前面最近的音乐源码行；如果该音乐行已经被一条歌词匹配，额外的 `w:` 会保留在 `Voice.lyricLines`，但不会覆盖已渲染的逐音歌词。没有后续 `w:` 的音乐行视为前奏、间奏或纯器乐行，不显示歌词。`w:` 以空白分隔音节；普通音节消耗一个发音单元并显示，`*` 消耗一个发音单元但不显示文字，可用于某些音无歌词的情况。

歌词消耗的是“发音单元”，不是原始事件数量：普通 note、附点 note、带时值/变音/八度的 note 都是一个发音单元；后续 `-` 延音、tie 连接的后续音符、圆滑线包围的后续音符会并入前一个发音单元，不额外消耗音节。rest、`[K:...]`、反复导航标记、小节线和 ending 不消耗歌词。三连音 `(3` 本身不消耗歌词；若三连音内没有 tie 或圆滑线，每个 note 各自消耗一个音节。所有原始歌词行仍完整保存在 `Voice.lyricLines`。

## 错误行为

`parseJabc` 不因用户输入错误而抛异常，而是返回 `{ success: false, errors }`。每项错误包含行号、列号、附近文本、错误 token（如适用）和修复建议。解析器会收集整份输入中能够识别的多个错误。

## 尚未支持

通用连音比例、嵌套反复导航、跨声部圆滑线、声部属性语义、同谱表声部合并及多段歌词复杂对齐尚未实现。新增任何语法时，必须同步更新本文档、README 示例和解析器测试。
